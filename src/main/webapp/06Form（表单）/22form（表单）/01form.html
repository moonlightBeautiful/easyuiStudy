    选项：
  1. url：string。要提交的表单动作 URL。
  2. onSubmit param 提交前触发，返回 false 来阻止提交动作。
  3. success data 当表单提交成功时触发。
  4. onBeforeLoad param 发出请求加载数据之前触发。返回 false 就取消这个动作。 onLoadSuccess data 当表单数据加载时触发。 onLoadError none 加载表单数据时发生某些错误的时候触发。
    事件：完全扩展自combo(自定义下拉框)和treegrid(数据表格)。
    方法：扩展自组合（combo）
  1. submit options 做提交动作，options 参数是一个对象，它包含下列属性：url：动作的 URL onSubmit：提交之前的回调函数 success：提交成功之后的回调函数
  2. load data 加载记录来填充表单。data 参数可以是一个字符串或者对象类型，字符串作为一个远程 URL，否则作为一个本地记录。
  3. clear none 清除表单数据。 reset none 重置表单数据。该方法自版本 1.3.2 起可用
  4. validate none 进行表单字段验证，当全部字段都有效时返回 true 。该方法和 validatebox 插件一起使用。
  5. enableValidation/disableValidation none 启用/禁用验证。该方法自版本 1.3.4 起可用。默认为开启验证。
    提示：
  1. 解决在IE下，form返回的数据是下载而不是异步加载：原因form异步返回的是json类型，IE为默认下载，更改响应ContentType
